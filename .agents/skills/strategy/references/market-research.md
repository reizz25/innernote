# Market Research

Use when studying a market, segment, use case, competitor set, or product opportunity.

This reference uses a product-strategy lens: a market is attractive when timing, demand, supply, scale effects, and organizational capability create a high-ROI entry point.

## Workflow

1. Define the decision: build, position, price, segment, launch, or defer.
2. Identify target user and job-to-be-done.
3. Gather current evidence from:
   - product docs, product state, and operational/provider context
   - AnyHost context when the repository is bound
   - customer/user language if available
   - public competitor pages
   - current market/product references
4. Compare alternatives by user job, not just feature lists.
5. Convert findings into product implications.

## Market Size

Estimate market size before over-optimizing a product idea. Do not stop at the current visible category; ask what larger demand could be unlocked if cost, convenience, trust, or supply changes.

Look for:

- current category size and growth
- adjacent categories that solve the same job
- user work currently done manually or internally that could move to a product/service
- infrastructure shifts that reduce cost or friction
- comparable markets in other geographies or verticals
- constraints that cap the opportunity

Failure modes:

- overestimating because a need feels real
- underestimating because the current category is small
- failing to update the estimate as new evidence arrives

## Scale Effects

Ask whether the opportunity improves structurally with scale.

Positive scale effects:

- more users improve value, liquidity, data, trust, or brand
- more supply improves selection, reliability, speed, or price
- more data improves matching, automation, ranking, risk control, or personalization
- larger scale improves capital access or operating leverage

Anti-scale effects:

- management complexity
- local operations burden
- quality degradation
- coordination cost
- support, abuse, trust, or compliance burden

A strong market has scale effects that compound faster than anti-scale effects.

## Market Dynamics

Check these dimensions:

| Dimension | Questions |
| --- | --- |
| Concentration | Is the market winner-take-most, fragmented, or structurally local? |
| Value chain | Where is value captured? Which layer has leverage? |
| First vs late mover | Is the category already educated? Did earlier attempts reveal traps? |
| Incremental vs replacement | Is this new demand, replacement demand, or a shift in how demand is served? |
| Frequency | Is the use case high-frequency enough to create habit, data, or distribution? |
| Timing | Why now? What changed in technology, cost, supply, demand, regulation, or behavior? |

Study "successful failures": products or companies that looked promising, got usage, or shaped the category but did not win. They often reveal demand, timing, cost, distribution, or organization traps.

## Supply And Demand

For operational, marketplace, infrastructure, and local-service products, diagnose the scarce side.

If supply is scarce, improve supply. If demand is scarce, improve demand. Watch for teams avoiding the scarce side because it is harder to work with.

| Dimension | Current state | Scarce side | Product implication |
| --- | --- | --- | --- |
| Demand | | | |
| Supply | | | |
| Geography | | | |
| Time | | | |
| Segment | | | |
| Non-market constraints | | | |

Moving a market online can change supply-demand balance: it can expand supply radius, reduce search cost, break local scarcity, create oversupply, and force a new operating model.
