# Prioritization

Use when choosing between product bets, roadmap items, market entry points, operating moves, or experiments.

## Core Frame

A strategy is an insight plus an action plan. The best strategy is the highest-ROI action for a specific time and space.

Time and space matter:

- market timing
- geography
- product segment
- user segment
- supply-demand state
- organization capability
- capital and attention constraints

Do not ask only "is this good?" Ask "is this the best use of scarce resources now?"

## ROI Inputs

Evaluate each option:

| Factor | Question |
| --- | --- |
| Market | How large can this become if it works? |
| Timing | Why now? Is the window opening or closing? |
| PMF | Is there a sharp wedge? |
| Scale effects | Does success compound? |
| Scarce side | Does it address the scarce side of supply/demand? |
| Organizational fit | Can this team execute it now? |
| Learning value | Will it reduce a key uncertainty quickly? |
| Risk | What can break trust, reliability, cost, or focus? |

## Stage-Aware Validation

Match the validation method to product maturity. A/B experiments need enough users, repeated behavior, and reliable instrumentation. Before that, the highest-ROI move is usually sharper research, not statistical testing.

| Stage | Main uncertainty | Better validation |
| --- | --- | --- |
| Idea / pre-launch | Is the need real and urgent? | interviews, market evidence, prototype, landing page/fake-door |
| Early usage | Can users reach the key value moment? | usability tests, concierge tests, onboarding observation, funnel/log review |
| Growing usage | Which lever moves activation or retention? | cohort analysis, before/after reads, small controlled rollout |
| Sufficient traffic | Which variant causally improves outcomes? | A/B test with primary metric, guardrails, and decision rule |

Do not prioritize an A/B test just because it sounds rigorous. If the product is low-volume, low-frequency, or weakly instrumented, prioritize research that produces a better hypothesis.

## Current-Stage Focus

Use a current-stage North Star or OMTM to concentrate decisions. The metric should match the most important bottleneck now:

- acquisition when too few qualified users enter
- activation when new users do not reach key value
- retention when users try but do not return
- resurrection when dormant users have a new reason to return
- monetization when value exists but willingness to pay is unproven
- referral/network/supply effects when usage should compound

Prefer retention and activation before scaling acquisition unless there is already evidence of repeat value.

## First vs Late Mover

First-mover advantages:

- category ownership
- early learning
- strategic resources
- talent/capital attention

Late-mover advantages:

- category education already happened
- feasibility is proven
- previous failures reveal traps
- better technology, UX, distribution, or cost structure may now exist
- incumbents may have organizational blind spots

Do not reject a market just because someone entered first. Study whether late-mover advantages are now larger than first-mover advantages.

## Layered And Classified Operations

No single product or operating motion satisfies all needs. Segment both product and operations.

Use layered operations when different tiers need different products, pricing, support, reliability, or sales motions.

Use classified operations when different categories, user jobs, supply types, or environments need different playbooks.

Avoid one KPI that pushes the organization toward only head, only tail, only demand, or only supply. The wrong metric can make a team ignore the strategically important middle.
