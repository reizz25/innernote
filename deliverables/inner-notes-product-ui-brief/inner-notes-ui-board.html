Inner Notes

一本本地保存、可以批注和自动回顾的私人日记本

它不是后台系统，也不是 AI 咨询窗口。用户先安静写下真实内容，再用文档式批注整理想法；周期结束后，由 Codex 读取本地文件生成周/月回顾。

设计原则

日记正文是主角。 标签、评论、AI、todo 都只能退后一层。
用户掌控解释权。 右侧是自己的边注，不是默认被 AI 分析。
真实文件保存。 内容写入 Markdown + JSON，能备份、迁移、手动查看。
AI 只做周期回顾。 定时读本地存档，写回本地 review 文件。

本地文件与数据结构

Inner Notes 的核心承诺是：日记不是临时网页状态，而是真实文件。

日记正文

用户可读，长期保存。

/journals/2026/07 2026-07-02.md

元数据

保存标签、todo、评论、锻炼。

/journals/2026/07 2026-07-02.json

回顾

周期结束后生成。

/reviews/weekly 2026-W26.md 2026-W26.json
设计重点：用户应该知道文件在哪里，且即使离开应用，也能打开 Markdown 查看自己的日记。

AI / Codex 自动回顾流程

当前 AI 不负责每篇日记的评论和聊天。它主要在周期结束后，读本地日记，写本地回顾。

定时唤醒

每周一检查上一个完整自然周；跨月时检查上个月。

读取本地

只读 journals 里的 Markdown / JSON 文件。

判断是否值得写

内容太少就跳过，不硬凑复盘。

生成回顾

指出模式、盲点、隐藏优点、小行动。

写回本地

输出到 reviews，页面只负责展示。

视觉语言

色彩

深灰绿做主色，纸白做背景，情绪标签用低饱和糖果色。

字体层级

正文和标题最清晰，左右栏和底部功能放大到可读但不抢主角。

交互

工具跟随鼠标所在段落，不遮挡文字；功能出现要轻，不制造压力。

语气

少系统词、少技术词；像“慢慢写”，不是“请填写”。

不要偏离的方向

Inner Notes 首先是一本日记，不是任务管理系统、AI 咨询机器人、情绪打卡表、数据看板或内容生产工具。

判断一个新功能该不该做：它是否让用户更容易写下真实内容、回看自己、保留重要线索？如果只是让产品更“强”，但让日记更吵，就应该退后。
