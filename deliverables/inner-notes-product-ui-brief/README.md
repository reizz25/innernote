# Inner Notes 产品功能、动线与界面说明

这份说明面向没有背景信息的产品设计师、前端工程师或 AI 助手。目标是让对方快速理解 Inner Notes 是什么、为什么这样设计、现在有哪些核心功能，以及后续优化时不要偏离的方向。

## 1. 产品定位

Inner Notes 是一个本地优先的私人日记本。它的核心不是“任务管理”、不是“文档后台”、也不是“AI 咨询工具”，而是一个让用户安静写下真实内容、回看自己、保留关键批注，并在周期结束后看见模式的个人记录空间。

产品气质关键词：

- 私人
- 安静
- 可信
- 本地保存
- 不催促
- 不过度分析
- 像一本会被认真翻看的日记

一句话定位：

> Inner Notes 是一本本地保存、可以批注和自动回顾的私人日记本。

## 2. 为什么这样设计

用户不喜欢 Word 式的松散格式，也不想要后台管理系统式的日记工具。她希望日记像飞书文档一样统一、简洁、有序，但又不能像文档批改工具那样让人有压力。

所以设计原则是：

- 日记正文永远是主角。
- 功能入口要轻，不能让用户一打开页面就觉得自己要填写一堆表单。
- AI 不应该主动“审判”用户；它主要做周期回顾，不在每篇日记旁边不断分析。
- 用户可以像读旧日记一样，对自己的文字做批注、划线、评论。
- 所有重要数据必须写成本地真实文件，方便备份、迁移和手动查看。

## 3. 当前产品结构

桌面端采用三栏结构：

```text
┌──────────────┬──────────────────────────────┬──────────────┐
│ 左侧日记本   │ 中间一页日记                  │ 右侧边注      │
│              │                              │              │
│ 写今天这一页 │ 日期 / 标题                   │ 评论          │
│ 最近记录     │ 正文编辑区                    │ 随便聊几句    │
│ 自动回顾     │ 保存 / 删除                   │ 已解决评论    │
│ 更多设置     │ 今天的标记 / todo / 锻炼       │              │
└──────────────┴──────────────────────────────┴──────────────┘
```

### 左侧：日记本索引

左侧像一本日记本的目录，不像文件管理器。

包含：

- 产品名 Inner Notes
- 主按钮：写今天这一页
- 全部记录，按时间倒序排列
- 每篇日记的日期、星期、时间、短摘要、情绪标签
- 自动回顾入口，只展示已经完整结束的周期
- 更多区域：保存位置、备份、提醒等弱功能

设计意图：

- 让用户感觉自己在翻一本长期日记。
- 默认展示所有记录，不再需要复杂归档筛选。
- 自动回顾只在周期结束后出现，例如“上周回顾 · 6 月 22 日 - 6 月 28 日”。

### 中间：一页日记

中间是主写作区，应该像一页安静的纸。

包含：

- 日期与具体时间
- 主标题：今天想记下什么？
- 正文编辑器
- 保存 / 删除
- 今天的标记
- 今日 todo 与本月未完成 todo
- 每日锻炼

编辑器能力：

- 一天可以写多篇日记
- 支持富文本编辑
- 支持复制粘贴图片
- 选中文字后出现浮动工具栏
- 空白段落前出现 `+` 插入菜单
- 有文字的段落前出现 `T` 段落格式菜单

设计意图：

- 写作体验接近飞书文档，但更轻、更私人。
- `+` 和 `T` 是文档编辑辅助，不应该遮挡文字。
- 格式能力服务于表达，不应该把日记变成排版软件。

### 右侧：边注与随笔

右侧不是 AI 聊天窗口，而是用户读自己日记时的边注区。

包含：

- 评论：选中正文后留下批注
- 评论可以点击“解决”后隐藏
- 随便聊几句：写旧日记触发的想法、补充、反驳、复盘

设计意图：

- 更像飞书文档的侧边评论，而不是心理咨询窗口。
- 用户掌控解释权；不是 AI 默认分析用户。
- 对旧日记的反应可以被保存，但不污染正文。

## 4. 主要用户动线

### 动线 A：写一篇日记

1. 打开 Inner Notes
2. 点击“写今天这一页”
3. 在中间正文区输入内容
4. 必要时插入图片、引用、高亮块、分割线
5. 保存到本地 Markdown 和 JSON 文件
6. 左侧自动出现这篇日记

### 动线 B：像文档一样整理正文

1. 鼠标移到正文某一行
2. 如果这一行为空，行外显示 `+`
3. 点击 `+`，可插入标题、列表、引用、高亮块、分割线、图片
4. 如果这一行有文字，行外显示 `T`
5. 点击 `T`，可切换正文、H1、H2、H3、编号列表、项目符号、引用
6. 再点一次同一格式，恢复自然正文

### 动线 C：给自己的文字批注

1. 选中正文中的一句话
2. 出现浮动工具栏
3. 可以加粗、斜体、改颜色、高亮，或点击“评论”
4. 评论显示在右侧
5. 评论右上角点击“解决”，评论隐藏

### 动线 D：记录当天状态

1. 写完正文后，用户可以展开“今天的标记”
2. 记录体重、睡眠、手机时间
3. 选择情绪标签
4. 添加今日 todo
5. 查看本月未完成 todo
6. 勾选每日锻炼动作

### 动线 E：自动周/月回顾

1. Codex 定时任务在周期结束后运行
2. 读取本地日记文件
3. 直接基于日记内容生成回顾，不调用 OpenRouter
4. 写回本地 `reviews` 文件夹
5. Inner Notes 左侧出现回顾入口
6. 用户点击后，在中间主区域阅读回顾

## 5. 本地文件系统

Inner Notes 的核心承诺是本地保存。

默认目录：

```text
/Users/zz/Desktop/InnerNotes
├── journals
│   ├── 2026
│   │   ├── 06
│   │   │   ├── 2026-06-22.md
│   │   │   ├── 2026-06-22.json
│   │   │   └── ...
│   │   └── 07
│   └── assets
│       └── 2026
└── reviews
    ├── weekly
    │   ├── 2026-W26.md
    │   └── 2026-W26.json
    └── monthly
        ├── 2026-06.md
        └── 2026-06.json
```

Markdown 文件保存用户可读正文。JSON 文件保存结构化数据，例如情绪、指标、todo、评论、锻炼状态、更新时间等。

## 6. AI 与自动回顾

当前产品中的 AI 不再负责每篇日记旁边的聊天和评论。AI / Codex 主要用于周期性回顾。

固定流程：

```text
Codex 定时任务
  ↓
读取本地 journals 文件
  ↓
判断上一个完整自然周 / 自然月是否值得回顾
  ↓
直接解读内容
  ↓
生成 markdown + json
  ↓
写入 reviews/weekly 或 reviews/monthly
  ↓
Inner Notes 页面展示回顾入口
```

周回顾内容建议包含：

- 这一周的主线
- 重复出现的模式
- 可能没看见的地方
- 隐藏优点
- 恢复方式
- 下周小动作
- 可以问自己的问题

语气原则：

- 指出模式，不审判用户
- 提出问题，不控制用户
- 给小行动，不灌鸡汤
- 温暖、直接、真诚

## 7. 当前 UI 风格

视觉目标：

- 安静、有日记感
- 轻微手帐贴纸感
- 不像后台系统
- 不像心理测评表
- 不像客服聊天窗口

色彩：

- 主色：深灰绿，用于主按钮和选中态
- 背景：纸白 / 暖灰
- 情绪标签：低饱和浅绿、灰蓝、奶油橙、淡紫
- 危险操作：低饱和红，只在需要时出现

字体层级：

- 主标题最大
- 正文清晰舒展
- 左右栏和底部功能比正文小一档，但不能太弱
- 辅助信息略浅，但必须可读

## 8. 后续优化重点

如果继续优化，优先级建议如下：

1. 打磨正文编辑器的 `+` / `T` 交互，让它像飞书文档一样贴近鼠标行，但不遮挡文字。
2. 完善图片粘贴、图片存储和 Markdown 引用路径。
3. 优化右侧评论区，让评论卡片更像边注，不像系统面板。
4. 改进周/月回顾的展示，让回顾像一篇真正写给用户的复盘，而不是报告。
5. 强化本地文件可见性：打开文件夹、立即备份、保存失败提示。
6. 做移动端适配：手机上应优先显示正文，左侧目录和右侧边注折叠成抽屉。

## 9. 不要偏离的方向

不要把 Inner Notes 做成：

- 任务管理系统
- 文档后台
- AI 咨询机器人
- 情绪打卡表
- 数据看板
- 内容生产工具

它首先是一本日记。其他功能都只是让这本日记更容易被写下、被回看、被温柔但清醒地理解。
