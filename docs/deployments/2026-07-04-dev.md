## Change

Prepared Innernote for AnyHost dev deployment with a Dockerfile, health endpoint, and Postgres-backed journal storage when `DATABASE_URL` is present. Local file storage remains the fallback when `DATABASE_URL` is absent.

Added a migration command for moving local journal JSON and review summaries into Postgres:

```sh
npm run migrate:journals
```

## Risks

- Dev is now running against AnyHost Postgres and the migrated note data has been verified through the public API.
- Image assets still use local filesystem storage. Notes text and review summaries are database-ready; pasted image persistence should move to object storage before relying on it in production.
- Future migrations that use external database profiles should preserve grants for the app runtime role or run through a migration role that owns/grants app tables by default.

## Follow-Up

1. Move pasted image persistence from local filesystem to object storage before production use.
2. Add a repeatable migration/grant step for future database changes.
